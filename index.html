Inheritance by using __proto__

Pros and cons of using __proto__

Pro Con
  • Setting __proto__ to not-object doesn't throw exception
  • Good browsers support (all non-microsoft browsers and IE11+)
  • Low performance
  • IE11 and below

Inheritance by using Object.setPrototypeOf()

Pros and cons of using Object.setPrototypeOf()

Pro Con
  • Allowance of the dynamic manipulation of an objects prototype
  • Possibility to force a prototype on a prototype-less object
  • Good browsers support (all non-microsoft browsers and IE10+)
  • Low performance
  • IE9 and below

Inheritance by using prototype property and constructor function

Pros and cons of using prototype property and constructor function

Pro Con
  • Good perfomance
  • Direct setting of __proto__ in an one-time-only way so browser can better optimize the object
  • Possibility to create objects without prototype
  • Good browsers support (all non-microsoft browsers plus IE9 and up)
  • If code is not yours, it could lead to posiible problems
  • IE5.4 and below

Inheritance by using Object.create()

Pros and cons of using Object.create()

Pro Con
  • Good perfomance
  • Fast, standard and optimizable
  • If it's all your code, you'll benefit
  • Almost full support (only IE5.5 and below amiss)
  • Lower perfomance with increasing of arguments (descriptors)
  • IE8 and below
